# Agentic 知识发现与发散探索计划

## 1. 目标

LazyMind 已具备知识库发现、文档列表、统计、语义检索、关键词检索、上下文扩展、多知识库检索、图片结果和引用能力，知识库可访问范围校验也已完成。

下一阶段的目标不再是“让 Agent 能调用知识库”，而是让 Agent：

1. 在用户没有预选知识库时，自主找到正确的知识库和文档。
2. 面对模糊、宽泛或探索性问题时，主动拆解方向、扩展概念并跨库寻找证据。
3. 根据中间发现调整搜索范围和研究方向，而不是一次检索后立即作答。
4. 识别多来源之间的互补、冲突、版本差异和知识缺口。
5. 在知识变化后增量更新研究结论，形成持续知识工作。

前端选择知识库仍然保留，但从“必填路由”调整为用户对 Agent 的可选约束：

- 未选择：允许 Agent 在当前用户可读范围内自主发现。
- 优先使用：先搜索用户所选范围，证据不足时可以扩展。
- 仅使用：严格限定在用户选择范围。
- 禁用：本轮不使用知识库。

## 2. 场景与能力域

现有及目标场景归纳为六个能力域：

| 能力域 | 典型场景 | 当前判断 |
|---|---|---|
| 定向检索 | 指定知识库问答、指定 PDF 查词、父节点和窗口扩展 | 基础完整，需优化工具选择和充分度判断 |
| 自主发现 | 不选知识库、按主题找库、相似知识库消歧、自动切换 | 已有工具入口，缺少稳定的语义路由 |
| 跨库综合 | 比较部门制度、综合案例与反馈、跨库研究报告 | 能检索，缺少结构化研究计划和来源平衡 |
| 发散探索 | 扩展相关主题、寻找遗漏维度、机会、反例和邻近场景 | 主要缺口，需要多步探索循环 |
| 时态与冲突 | 最新制度、历史规定、版本比较、冲突识别 | 需要版本链、生效时间和冲突模型 |
| 持续知识工作 | 监控新增内容、结论失效通知、FAQ 和知识缺口 | 需要知识事件和结论—证据依赖关系 |

## 3. 三阶段路线

| 阶段 | 核心问题 | 主要交付 | 用户价值 |
|---|---|---|---|
| [阶段一：自主找到知识](./phase-1-autonomous-discovery.md) | 知识在哪里 | KB 画像、语义路由、范围状态、自动切换、路由评测 | 用户不必先理解并选对知识库 |
| [阶段二：围绕问题发散探索](./phase-2-divergent-exploration.md) | 还应该知道什么 | Exploration Plan、多分支检索、查询扩展、充分度、冲突与缺口 | 宽泛问题也能形成系统性洞察 |
| [阶段三：持续研究与知识演进](./phase-3-continuous-research.md) | 知识变化后怎么办 | 版本时态、知识事件、依赖图、增量研究、知识沉淀 | 从一次性问答升级为持续知识工作 |

三个阶段必须顺序推进：

1. 阶段一的路由准确性、结构化范围和 Trace 是发散探索的基础。
2. 阶段二的研究计划、分支状态和证据依赖是增量研究的基础。
3. 阶段三才引入版本、事件和受控知识沉淀，避免过早建设复杂知识图谱。

## 4. 统一设计原则

### 4.1 自主不等于无边界

Agent 可以自主选择候选知识库、搜索词和探索方向，但必须遵守用户指定范围、检索预算、任务深度和停止条件。

### 4.2 分离发现、检索和探索

- `discover`：寻找相关知识库、文档和主题。
- `retrieve`：围绕一个明确意图获取证据。
- `explore`：根据已有证据形成新假设和检索分支。

不要让一个 `kb_search` 同时承担路由、检索、发散和总结。

### 4.3 发散必须有来源

新增探索方向必须来自用户目标、证据实体、覆盖缺口、冲突检查、反例验证或相邻场景，不为了“显得全面”无限扩展。

### 4.4 证据与推断分离

回答明确区分原文事实、跨来源综合、Agent 推断和当前无法验证的部分。

### 4.5 全链路可观测和可评估

除最终答案外，还要记录和评估候选知识库、选择理由、query 改写、探索分支、证据充分度、停止原因和最终引用。

## 5. 总体目标体验

用户可以直接提出：

> 我们过去做过哪些面向制造业的智能化项目？有哪些经验可以复用？还可能有哪些新机会？

Agent 应自行：

1. 找到项目案例、售前资料、复盘记录等候选知识库。
2. 拆分项目清单、成功模式、失败模式和机会方向。
3. 分别检索并根据中间证据扩展相关产品、行业和场景。
4. 区分直接事实、归纳模式和机会假设。
5. 输出引用、搜索范围、冲突、未知项及可继续探索的方向。

## 6. 统一核心指标

- KB 路由 Top-1 准确率和 Top-3 召回率。
- 无预选知识库时的成功检索率。
- 关键问题维度覆盖率。
- 有效探索分支比例。
- 独立来源数量和来源多样性。
- 重复、无价值检索比例。
- 引用有效率和结论证据覆盖率。
- 冲突、过期证据和知识缺口发现率。
- 用户手动改选知识库或删除探索分支的比例。
- 单次研究与增量研究的成本和延迟。

## 7. 推荐实施顺序

1. 知识库画像与语义目录。
2. 知识空间搜索和结构化检索范围状态。
3. 无前端预选的端到端 Agent 测试。
4. 前端知识范围模式与使用范围展示。
5. 路由 Trace、评测集和线上指标。
6. Exploration Plan、证据充分度和停止条件。
7. 查询扩展、多分支并行、反例与冲突探索。
8. 版本时态、知识事件、依赖图和持续研究。

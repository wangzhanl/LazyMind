# 阶段 6：插件创建与 StateGraph 在线编辑器

> 支持从现有 Skill（SKILL.md）一键生成插件的 scenario 与状态机草稿，用户可在前端通过图形化或 YAML 编辑器交互调整后发布；同时支持对已有插件的状态机进行在线修改并热重载生效。

---

## 零、从 Skill 生成插件（创建入口）

1. **Skill → 插件草稿**：用户可从本地已有 Skill 或通过 URL 引入网络上的 Skill（SKILL.md 格式），系统调用 AI 分析其工作流描述，自动生成对应的插件草稿，包含：`scenario.md`（意图识别描述）、`state.yml`（步骤列表与状态机转移）、`plugin.yaml`（插件元数据与 UI slot 声明）。
2. **草稿预览与交互修改**：生成的草稿在前端编辑器中展示，用户可在发布前对步骤名称、转移条件、slot 配置等进行交互式调整（使用下方图形化编辑器或 YAML 编辑器）。
3. **发布为插件**：用户确认草稿后一键发布，插件立即注册到系统并对当前会话可用，无需部署操作。
4. **从零新建**：不依赖已有 Skill，支持用户直接在编辑器中从空白画布新建插件，手动定义步骤与状态机。

---

## 一、权限与访问控制

1. **管理员限定**：StateGraph 编辑器仅对具有管理员角色的用户开放；在开发调试环境下可通过配置允许非管理员访问。
2. **只读模式降级**：普通用户访问时，StateGraph 仅以只读模式展示（同阶段 5），不显示编辑入口。

## 二、状态机读写接口

3. **读取接口**：提供接口返回插件当前状态机定义，包含：图结构（节点、边及其属性）、节点在画布上的位置布局、原始 YAML 文本。
4. **保存接口**：提供接口接受修改后的状态机定义；保存前先执行合法性校验（环路检测、孤立节点、缺失 `__start__`/`__end__`、condition 为空等），校验失败返回详细错误列表；校验通过后更新状态机定义并立即热重载生效，无需重启服务。
5. **持久化存储**：状态机定义存入数据库，不依赖容器本地文件系统，保证重启和多副本部署下的一致性；原有文件作为初始导入来源，导入后以数据库为准。

## 三、图形化编辑

6. **节点编辑**：支持新增、删除步骤节点；可编辑步骤 ID（唯一键）、显示标签、步骤默认 human/auto 模式。
7. **边编辑**：支持在节点间连线新增转移边、删除已有边；可内联编辑转移条件文本。
8. **布局持久化**：节点拖拽后的位置自动保存，下次打开编辑器时恢复上次布局；不同插件的布局独立存储。
9. **保存校验反馈**：点击保存时若校验未通过，在编辑器内直接展示错误列表（标注出问题的节点/边），不关闭编辑器；用户修正后可再次保存。

## 四、YAML 双向编辑

10. **YAML 预览面板**：编辑器内提供 YAML 文本视图，图形编辑时实时同步更新对应 YAML 内容；用户可直接在 YAML 视图中修改，修改后图形视图自动更新。
11. **两种视图保持同步**：图形视图与 YAML 视图始终反映同一份状态机定义，切换视图时不丢失未保存的修改。

## 五、热重载

12. **修改即时生效**：状态机保存成功后，运行中服务无需重启即可使用新的状态机定义；新会话立即采用新定义，不影响正在进行中的现有会话（继续使用触发时的状态机版本）。

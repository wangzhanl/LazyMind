# 阶段 5：监控看板与流程可视化

> 为用户和管理员提供插件执行过程的可视化监控能力：任务看板实时展示会话内所有步骤的执行状态，StateGraph 以图形方式呈现插件工作流结构与当前进度。

---

## 一、任务看板

1. **看板数据接口**：提供接口返回当前会话下所有已触发的插件执行记录，包含插件名称、当前进行步骤、各步骤状态（pending / running / waiting / succeeded / failed）及 artifact 产出摘要。
2. **任务卡片列表**：前端看板展示所有插件执行记录，每张卡片显示插件名、当前步骤、整体进度；步骤状态变更时通过已有的 Conversation Events SSE 实时刷新，无需轮询。
3. **跳转到插件面板**：点击看板卡片可跳转到对应 Plugin Session 的 Plugin Panel，直接查看 artifact 内容。
4. **历史会话看板**：已完成或失败的插件执行记录也在看板中展示，支持按状态筛选（进行中 / 已完成 / 失败）。

## 二、StateGraph 可视化（只读）

5. **状态图数据接口**：提供接口返回插件状态机的图结构叠加当前运行状态，包含：
   - 节点：步骤 ID、显示名称、当前状态（未开始 / 运行中 / 等待确认 / 成功 / 失败）、已产出 artifact 摘要
   - 边：转移条件文本、是否为从当前节点出发的合法后继路径
6. **StateGraph 只读渲染**：前端以有向图方式展示工作流；节点显示步骤名和状态 badge；边显示转移条件（悬停展示全文）；从当前步骤出发的合法后继边以高亮或虚线区分；节点可展开查看 artifact 摘要。
7. **集成到插件面板**：StateGraph 作为 Plugin Panel 侧边栏或 Tab 的一部分展示，与 Plugin Panel 主体内容同时可见，用户无需切换页面。
8. **实时状态刷新**：步骤状态发生变更时（通过 Conversation Events SSE），StateGraph 中对应节点的状态 badge 自动更新，无需手动刷新页面。
